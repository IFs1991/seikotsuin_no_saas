## 3. Frozen hosted workload and SLO

The proposal uses a local Node 24 server-side harness against the isolated hosted Supabase project. No production Vercel project or public staging deployment is used.

Execution order is frozen:

1. 300-second unscored warm-up at concurrency 1.
2. 600-second `sample_1_read_heavy_c50`.
3. 600-second `sample_2_mixed_crud_c50`.
4. 600-second `sample_3_read_heavy_repeat_c50`.

Each scored sample runs a closed loop of 50 virtual users, ramps all users within 30 seconds, and then applies no artificial think time. Requests use deterministic weighted round-robin allocation from the frozen per-sample mix. An expected denial counts as successful only when HTTP status, SQLSTATE, row count, ACL verdict, and RLS verdict all match the frozen case; every mismatch is a failure. Mixed CRUD is synthetic and must clean up only its own qualification-owned rows.

Every scored sample and the pooled result must satisfy:

| Metric              |                   Frozen proposal |
| ------------------- | --------------------------------: |
| p95                 |                     `<= 2,000 ms` |
| p99                 |                     `<= 3,000 ms` |
| throughput          | `>= 20 completed requests/second` |
| unexpected failures |                               `0` |
| 5xx rate            |                              `0%` |
| timeout rate        |                              `0%` |
| request timeout     |                      `10 seconds` |

Database abort thresholds are CPU `>75%` for 300 seconds, pool headroom `<25%` for 120 seconds, any lock wait `>1,000 ms`, cumulative WAL across all three scored samples `>268,435,456 bytes`, or full migration replay `>900 seconds`. CPU/pool are sampled every 60 seconds, lock state every 5 seconds, and WAL at every sample boundary. Each sample uses nearest-rank percentiles and completed requests divided by 600 seconds; pooled percentiles use all completed requests without reweighting and pooled throughput divides total completed requests by 1,800 seconds.

No warm-up or scored sample may replace another sample. No sample may be discarded, reordered, or reclassified. The hosted SLO is additive and cannot replace any fixed PR11 execution, WAL, plan, or semantic gate. The full machine contract is [hosted-slo-contract.proposed.json](evidence/commercial-hardening/pr12/hosted-slo-contract.proposed.json).
