| 項目 | 内容 |

## 1. 目的

Tiramisuの予約管理・シフト入力・ブロック管理において、現場スタッフが直感的に理解できる横向きタイムテーブルUIを提供する。

本仕様の目的は以下である。

- 日本の医療・治療院現場で見慣れた横向き時間割に近いUIにする
- 予約、シフト、休憩、外部予定、ブロックを同一設計思想で扱う
- スタッフ別・ベッド別・リソース別の稼働状況を一画面で把握できるようにする
- 予約事故、二重予約、シフト入力ミスを減らす
- 将来的なGoogleカレンダー連携、LINE予約、SALON BOARDブロック同期の土台にする

---

## 2. 基本方針

### 2.1 横向きタイムテーブルを採用する

TiramisuのカレンダーUIは、以下を基本とする。

```text
縦軸: スタッフ / ベッド / リソース / 外部スケジュール
横軸: 時刻
カード: 予約 / シフト / 休憩 / ブロック / 外部予定
```

Google Calendarのような縦方向の時間軸ではなく、日本の院内予約表・紙のシフト表に近い横方向の時間軸を採用する。

### 2.2 正本はTiramisu DBとする

カレンダーUIはGoogle Calendar風にしてよいが、予約・シフト・ブロックの正本はTiramisu DBとする。

```text
予約SSOT = Tiramisu reservations
シフトSSOT = Tiramisu shifts
ブロックSSOT = Tiramisu blocks
Google Calendar = 外部表示 / 補助同期
```

Google Calendar APIを予約台帳・シフト台帳の正本にしない。

### 2.3 操作より視認性を優先する

初期MVPでは、高度なドラッグ操作より以下を優先する。

- 見やすいこと
- 誤クリックしにくいこと
- 予約枠・休憩・ブロックの違いが明確なこと
- スタッフ別/ベッド別の空き状況が一目で分かること
- 詳細パネルで操作を確定できること

## 3. 対象画面

### 3.1 予約管理カレンダー

対象ルート候補:

```text
/reservations
/reservations/calendar
```

主な目的:

- 予約一覧表示
- 予約作成
- 予約編集
- 予約キャンセル
- スタッフ別予約確認
- ベッド/リソース別予約確認
- ブロック確認

### 3.2 シフト入力カレンダー

対象ルート候補:

```text
/staff/shifts
/staff/schedule
```

主な目的:

- スタッフ勤務時間の入力
- 休憩時間の入力
- 休み入力
- 外部予定入力
- 繰り返しシフト設定
- Google Calendar連携状態の確認

### 3.3 ブロック管理

対象ルート候補:

```text
/blocks
/reservations/blocks
```

主な目的:

- 予約不可枠の作成
- スタッフ不在枠の作成
- ベッド/リソース使用不可枠の作成
- 会議/研修/院内イベントの登録

## 4. 画面レイアウト

## 4.1 共通レイアウト

```text
┌────────────────────────────────────────────────────────────┐
│ Header: title / date navigation / view switch / actions     │
├──────────────┬───────────────────────────────┬─────────────┤
│ Sidebar      │ Horizontal Timetable           │ Detail Panel │
│ Navigation   │                               │             │
└──────────────┴───────────────────────────────┴─────────────┘
```

### Header

表示要素:

- 画面タイトル
- 今日ボタン
- 前日/翌日 または 前週/翌週ボタン
- 日付/期間表示
- 表示切替
  - 日
  - 週
  - 月
- 表示対象切替
  - スタッフ
  - ベッド
  - スタッフ + ベッド
- 検索
- フィルタ
- 作成ボタン

### Main Timetable

表示要素:

- 横軸: 時刻
- 縦軸: スタッフ/リソース
- カード: 予約/勤務/休憩/ブロック
- 現在時刻ライン
- 営業時間外の薄い背景
- 休診日/休日の背景

### Detail Panel

右側に選択中の予約・シフト・ブロックの詳細を表示する。

原則:

- 一覧上で直接すべて編集させない
- カード選択 → 詳細パネル → 編集/保存
- 破壊的操作は確認を挟む

# 5. 予約管理カレンダー仕様

## 5.1 表示構造

### 基本構造

```text
横軸: 09:00 - 21:00
縦軸: スタッフ / ベッド / リソース
カード: 予約 / ブロック / 休憩 / 外部予定
```

### セクション

予約管理画面では、以下のセクションを表示できる。

1. スタッフ
2. ベッド/施術室
3. 外部予定
4. ブロック

MVPでは以下を推奨する。

```text
上段: スタッフ別
下段: ベッド/リソース別
```

## 5.2 予約カード

### 表示内容

予約カードには以下を表示する。

| 項目 | 表示 |
|---|---|
| 時刻 | 必須 |
| 患者名 | 必須 |
| メニュー | 必須 |
| 担当者 | 任意 |
| ステータス | 色/バッジ |
| 初回/再来 | 任意 |
| チャネル | 任意 |

### 表示例

```text
09:00 - 09:50
山本 一郎 様
骨盤矯正
山田 太郎
```

## 5.3 色分け

| 種別 | 色方針 |
|---|---|
| 初診/カウンセリング | 淡い青 |
| 再診 | 淡い緑 |
| 施術/治療 | 淡い橙 |
| 検査/処置 | 淡い紫 |
| 点滴/注射 | 淡いピンク |
| 休憩 | グレー |
| 外部予定 | 斜線 + グレー/紫 |
| ブロック | 斜線 + グレー |
| キャンセル | 低彩度 + 取り消し線または非表示切替 |

### 注意

色だけに依存しない。必ずラベルまたはアイコンも併用する。

## 5.4 操作

### カードクリック

予約カードをクリックすると右側に予約詳細を表示する。

右側パネルの内容:

- ステータス
- 患者名
- 電話番号
- 日付
- 時刻
- メニュー
- 担当者
- ベッド/リソース
- メモ
- 内部メモ
- 編集ボタン
- キャンセルボタン

### 空き枠クリック

空き枠クリック時の挙動:

```text
空き枠クリック
↓
予約作成パネルを開く
↓
クリックした時刻・スタッフ・リソースを初期値に入れる
```

MVPではクリック作成でよい。ドラッグ作成は後回しでもよい。

### ドラッグ操作

将来的には以下をサポートする。

- 予約時間変更
- 担当者変更
- ベッド変更

ただし、MVPでは必須ではない。

ドラッグ変更時は必ず以下を行う。

- 衝突チェック
- 確認モーダル
- 通知送信有無の確認
- 監査ログ記録

## 5.5 衝突表示

予約衝突がある場合は以下を表示する。

- カード枠を赤にする
- 警告アイコン表示
- 詳細パネルに衝突理由表示
- 保存不可

衝突例:

- 同一スタッフの同時間帯予約
- 同一ベッドの同時間帯予約
- ブロック時間帯への予約
- シフト外予約
- 休憩時間への予約

# 6. シフト入力カレンダー仕様

## 6.1 表示構造

```text
横軸: 08:00 - 21:00
縦軸: スタッフ
カード: 勤務 / 休憩 / 休み / 外部予定 / 会議 / 研修
```

### 推奨表示

- 日表示: 1日のスタッフ別シフト
- 週表示: スタッフごとの週シフト
- 月表示: 詳細ではなく集計中心

MVPでは **日表示 + 週表示** を優先する。

## 6.2 シフトカード

### 表示内容

| 項目 | 表示 |
|---|---|
| 時刻 | 必須 |
| 区分 | 必須 |
| メモ | 任意 |
| Google連携状態 | 任意 |
| 確定/下書き | 任意 |

### 表示例

```text
09:00 - 18:00
勤務
```

```text
13:00 - 14:00
休憩
```

## 6.3 区分

| 区分 | 意味 |
|---|---|
| work | 勤務 |
| break | 休憩 |
| day_off | 休み |
| external | 外部予定 |
| meeting | 会議 |
| training | 研修 |
| unavailable | 予約不可 |

## 6.4 色分け

| 種別 | 色方針 |
|---|---|
| 勤務 | 淡い緑 |
| 休憩 | 淡い橙 |
| 休み | 淡い赤/ピンク |
| 外部予定 | 淡い紫 + 斜線 |
| 会議/研修 | グレー/紫 |
| 未確定 | 点線枠 |
| 確定 | 通常枠 |

## 6.5 操作

### 空き枠ドラッグ

シフト入力では、予約管理よりもドラッグ操作を優先する。

```text
スタッフ行の空き時間をドラッグ
↓
シフト作成パネルを開く
↓
開始/終了時刻を自動入力
```

### カードクリック

シフトカードクリックで右側詳細パネルを開く。

表示項目:

- ステータス
- スタッフ
- 日付
- シフト時間
- 休憩時間
- 区分
- メモ
- 繰り返し
- Googleカレンダー連携状態
- 削除ボタン

### 繰り返し設定

MVPでは簡易対応。

- 繰り返さない
- 毎週同じ曜日
- 平日のみ
- カスタム

初期はUIだけ用意し、実装は段階的でよい。

# 7. ブロック表示仕様

## 7.1 ブロックの目的

ブロックは予約不可枠を表す。

例:

- 休憩
- 外部予定
- 会議
- 研修
- 院内イベント
- 設備メンテナンス
- ベッド使用不可
- スタッフ不在

## 7.2 表示場所

ブロックは以下に表示する。

- 予約管理カレンダー
- シフト入力カレンダー
- Blocks画面

## 7.3 表示ルール

- 斜線背景を使用する
- 通常予約と明確に区別する
- 理由を表示する
- 操作権限がない場合は閲覧のみ

## 7.4 SALON BOARD同期との関係

将来的にSALON BOARDへ同期する場合、最初に同期する対象は予約ではなくブロックとする。

```text
Tiramisu block
↓
Sync queue
↓
SALON BOARD RPA / Adapter
↓
同期結果をTiramisuへ返す
```

# 8. Google Calendar連携UI

## 8.1 基本方針

Google Calendarは外部同期先/同期元として扱う。

画面上では以下を表示する。

- 連携状態
- 連携カレンダー名
- 最終同期日時
- 同期方向
- 連携設定ボタン

## 8.2 表示例

```text
Googleカレンダー連携
● 連携中（メインカレンダー）
[連携設定を開く]
```

## 8.3 同期方向

| 同期方向 | 意味 | 初期推奨 |
|---|---|---|
| export_only | Tiramisu → Google Calendar | 推奨 |
| import_review | Google Calendar → Tiramisu候補 | 次段階 |
| two_way | 双方向同期 | 後回し |

MVPでは `export_only` を推奨する。

# 9. ビュー切替

## 9.1 日表示

目的:

- 当日の稼働確認
- 受付/院長の実運用
- 予約作成
- シフト入力

横軸:

```text
08:00 - 21:00
```

縦軸:

```text
スタッフ / ベッド / リソース
```

## 9.2 週表示

目的:

- 週単位の予約密度確認
- シフト計画
- スタッフ稼働比較

予約管理での週表示は、情報過多になりやすいため以下を検討する。

- 日付をタブで切り替える
- 1日単位横タイムテーブルを維持する
- 週表示では集計/密度表示に寄せる

シフト入力では週表示を優先してよい。

## 9.3 月表示

目的:

- 稼働日確認
- 休診日確認
- シフト未入力日確認
- 予約件数集計

月表示では詳細カードを詰め込みすぎない。

# 10. レスポンシブ方針

## 10.1 デスクトップ優先

本カレンダーUIは業務PCでの利用を主対象とする。

推奨幅:

```text
1280px以上
```

最適幅:

```text
1440px - 1920px
```

## 10.2 タブレット

タブレットでは以下を行う。

- 右詳細パネルをdrawer化
- サイドバーを縮小
- 横スクロール許容

## 10.3 スマートフォン

スマホでは同じタイムテーブルを無理に表示しない。

スマホ用には以下を提供する。

- 日別リスト
- 自分の予約一覧
- 自分のシフト一覧
- 詳細確認
- 簡易キャンセル/確認

---

# 11. アクセシビリティ

## 11.1 色以外の情報表示

カード種別は色だけでなく、必ずテキストまたはアイコンでも識別可能にする。

## 11.2 キーボード操作

最低限以下を満たす。

- Tabでカード移動
- Enterで詳細パネルを開く
- Escで詳細パネルを閉じる
- フィルタ/検索フォームへ移動可能

## 11.3 視認性

- 12px未満の文字を多用しない
- 予約カード内の文字は最大3〜4行
- 長い患者名は省略し、詳細パネルで全文表示

---

# 12. 状態表示

## 12.1 ローディング

- タイムテーブルにはskeletonを表示
- 右詳細パネルは空状態にする

## 12.2 空状態

予約がない場合:

```text
この時間帯の予約はありません
空き枠をクリックして予約を作成できます
```

シフトがない場合:

```text
この日のシフトは未入力です
ドラッグしてシフトを作成できます
```

## 12.3 エラー状態

- APIエラー
- 権限エラー
- 衝突エラー
- 同期エラー

を区別して表示する。

# 13. パフォーマンス方針

## 13.1 仮想化

スタッフ数・リソース数が多い場合は、行のvirtualizationを検討する。

目安:

```text
表示行数 50行以上でvirtualization検討
```

## 13.2 データ取得

カレンダー初期表示では必要範囲のみ取得する。

```text
日表示: 当日分
週表示: 対象週分
月表示: 集計中心
```

## 13.3 再レンダリング削減

- カードコンポーネントをmemo化
- 詳細パネルを分離
- フィルタ状態をURL queryに同期
- 重い計算はuseMemo化

# 16. 受入基準

## 16.1 共通

- [ ] 横軸が時刻、縦軸がスタッフ/リソースになっている
- [ ] 予約/シフト/ブロックの色とラベルが区別できる
- [ ] 1280px幅で業務利用可能な視認性がある
- [ ] 右詳細パネルから主要操作ができる
- [ ] 権限がない操作は非表示またはdisabledになる
- [ ] エラー/空状態/ローディングが用意されている

## 16.2 予約管理

- [ ] 予約一覧が時間軸に沿って正しく表示される
- [ ] 同一スタッフの予約衝突が表示/保存時に検知される
- [ ] 同一リソースの予約衝突が表示/保存時に検知される
- [ ] キャンセルは物理削除ではなくstatus更新として扱われる
- [ ] ブロック時間帯に予約しようとすると警告される

## 16.3 シフト入力

- [ ] スタッフ別に勤務/休憩/休みが表示される
- [ ] シフト時間を編集できる
- [ ] 休憩時間を設定できる
- [ ] 外部予定/会議/研修を通常勤務と区別できる
- [ ] Google Calendar連携状態が表示される

# 17. 非推奨

以下は避ける。

## 17.1 Google Calendarを正本にする

避ける理由:

- 業務データが不足する
- clinic_id / role / RLS と合わない
- 予約衝突判定が不安定になる
- 売上/日報/患者管理と接続しにくい

## 17.2 縦型タイムテーブルのみで固定する

避ける理由:

- 日本の院内運用では横向き予約表の方が直感的なケースが多い
- スタッフ/ベッド比較がしづらい
- 紙の運用からの移行負荷が上がる

## 17.3 カードに情報を詰め込みすぎる

避ける理由:

- 視認性が落ちる
- 誤クリックが増える
- 詳細パネルの意味が薄れる

## 17.4 ドラッグ操作をMVP必須にする

避ける理由:

- 実装難度が上がる
- 衝突/通知/監査ログが複雑化する
- 初期品質が落ちやすい

# 18. 結論

TiramisuのカレンダーUIは、Google Calendarそのものを模倣するのではなく、日本の治療院・整骨院業務に合う横向きタイムテーブルとして設計する。

基本方針は以下で固定する。

```text
横軸 = 時刻
縦軸 = スタッフ / ベッド / リソース
カード = 予約 / シフト / 休憩 / ブロック / 外部予定
詳細操作 = 右側パネル
正本 = Tiramisu DB
Google Calendar = 外部同期/表示補助
```

この設計により、現場の紙予約表・紙シフト表に近い認知負荷で、Tiramisuの予約、シフト、ブロック、外部連携を統合できる。
